# CA Commer v2

## 项目目标（当前阶段）

本仓库当前阶段只做一件事：

👉 **搭平台骨架（而不是做完整系统）**

重点包括：

- 平台 Home 页结构
- 一级 / 二级导航设计
- 多角色 / 多 workspace 的入口方式
- 主题域划分（OTP / Contract / Billing 等）
- 页面层级关系（而不是页面细节）

---

## 本阶段设计原则

### 1. 不锁死页面内容
页面只定义：
- 目标
- 入口
- 层级
- 跳转关系

不定义：
- 详细字段
- 表格结构
- 复杂交互

---

### 2. 导航优先于页面
用户必须先看懂：

- 这个平台分几块
- 每一块是干什么的
- 我应该从哪里进入

---

### 3. Home 页优先
Home 页必须回答：

- 我是谁（角色）
- 我现在该关注什么
- 我下一步去哪

---

### 4. 一次只改一个模块
避免系统级混乱

---

## 当前阶段不做

- 不做完整业务逻辑
- 不做最终 UI
- 不做 API
- 不做复杂权限
- 不做细粒度字段设计

---

## 开发顺序

1. 平台 Shell
2. Home 页
3. 导航结构
4. 骨架页面
5. 单页逐步精修

---

## 工作方式（必须遵守）

- 必须先出 implementation plan
- 不允许直接写代码
- 不允许擅自扩展页面细节
- 不允许跨模块改动
